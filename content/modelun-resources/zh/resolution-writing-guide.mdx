决议是模拟联合国委员会的主要成果。它们代表委员会对某一问题的官方立场，并提出具体的行动建议。学习撰写有效的决议是你在模联中能够培养的最重要技能之一——优秀的决议展示领导力，吸引联盟伙伴，并增加获奖机会。

本指南涵盖从基本结构到撰写能够通过并产生影响的决议的高级技巧。

## 什么是决议？

决议是联合国委员会通过的正式文件，表达委员会的立场并就特定问题建议采取行动。在模拟联合国中，决议是你所有辩论、谈判和协作的切实成果。

### 主要特征

**(a)** 决议以单一连续的长句书写，各条款之间用逗号和分号分隔。

**(b)** 决议有特定结构：标题、序言性条款和执行性条款。

**(c)** 在提交讨论之前，决议需要起草国（作者）和附议国（支持者）。

**(d)** 一旦通过，决议代表委员会的官方立场。

## 决议结构

每份决议都遵循相同的基本结构。理解这一结构对于撰写有效文件至关重要。

### 标题

标题用于识别决议，包括：

**(a)** **委员会名称**：委员会的全称（如"联合国大会"）

**(b)** **议题**：所讨论的问题

**(c)** **起草国**：决议的主要作者（最多为委员会人数的5%，向上取整）

**(d)** **附议国**：希望看到决议被讨论的代表（最少为委员会人数的20%，向下取整）

**(e)** **决议标题**：可选的创意名称或首字母缩写

### 序言性条款

序言性条款为你的决议设定背景。它们解释为什么这个问题很重要以及委员会应该考虑什么背景。

#### 序言性条款的目的

**(a)** 引用相关的联合国过去行动、条约和决议

**(b)** 陈述关于情况的重要事实

**(c)** 承认问题的范围和严重性

**(d)** 为采取行动建立法律和历史基础

**(e)** 表达委员会对该问题的关切或赞赏

#### 格式

**(a)** 每个条款以*斜体*的现在分词（以-ing结尾的动词）开头

**(b)** 每个条款以逗号结尾

**(c)** 条款不编号

#### 常用序言性短语

| 短语 | 使用时机 |
|--------|-------------|
| *认识到（Acknowledging）* | 承认一个事实或情况 |
| *确认（Affirming）* | 确认一项原则或承诺 |
| *对...感到震惊（Alarmed by）* | 对严重情况表示关切 |
| *意识到（Aware of）* | 注意到重要的背景信息 |
| *铭记（Bearing in mind）* | 考虑相关因素 |
| *相信（Believing）* | 表达对某项原则的信念 |
| *有信心（Confident）* | 表达对某一结果的信任 |
| *深切关注（Deeply concerned）* | 强烈表达担忧 |
| *强调（Emphasizing）* | 突出重要性 |
| *充分意识到（Fully aware）* | 强调性地承认 |
| *以...为指导（Guided by）* | 遵循既定原则 |
| *审议了（Having considered）* | 在审查信息之后 |
| *注意到（Noting）* | 观察一个事实 |
| *观察到（Observing）* | 注意到一种情况或趋势 |
| *重申（Reaffirming）* | 重申对过去立场的承诺 |
| *承认（Recognizing）* | 承认一个重要事实 |
| *回顾（Recalling）* | 引用过去的决议或条约 |
| *考虑到（Taking into account）* | 考虑相关因素 |
| *欢迎（Welcoming）* | 对进展表示赞同 |

### 执行性条款

执行性条款是行动项目——委员会实际希望发生的事情。这是你决议的核心。

#### 执行性条款的目的

**(a)** 提出解决问题的具体行动

**(b)** 建立机制、项目或机构

**(c)** 要求成员国、秘书长或其他联合国实体采取行动

**(d)** 设定时间表和问责措施

#### 格式

**(a)** 每个条款以**加粗**或*下划线*的现在时动词（第三人称单数）开头

**(b)** 条款编号（1, 2, 3...）

**(c)** 子条款使用字母（a, b, c...），然后是罗马数字（i, ii, iii...）

**(d)** 每个条款以分号结尾，最后一个条款以句号结尾

#### 常用执行性短语

| 短语 | 强度 | 使用时机 |
|--------|----------|-------------|
| *呼吁（Calls upon）* | 强 | 敦促各国采取具体行动 |
| *决定（Decides）* | 最强 | 作出明确的委员会决定 |
| *要求（Demands）* | 非常强 | 紧急或危急情况 |
| *鼓励（Encourages）* | 中等 | 建议自愿行动 |
| *认可（Endorses）* | 中等 | 支持现有倡议 |
| *设立（Establishes）* | 强 | 创建新机制或机构 |
| *进一步邀请（Further invites）* | 较弱 | 额外建议 |
| *注意到（Notes）* | 较弱 | 承认但不采取行动 |
| *宣布（Proclaims）* | 强 | 发表声明 |
| *建议（Recommends）* | 中等 | 建议行动方案 |
| *请求（Requests）* | 中等 | 要求具体行动 |
| *强烈谴责（Strongly condemns）* | 非常强 | 表达严厉谴责 |
| *支持（Supports）* | 中等 | 支持现有努力 |
| *敦促（Urges）* | 强 | 推动重要行动 |

## 撰写有效条款

条款的质量决定了决议的质量。以下是撰写引人注目且有效的条款的方法。

### 序言性条款写作提示

**(a)** 尽可能引用具体的联合国决议编号（例如，"回顾大会第66/288号决议..."）

**(b)** 包含相关统计数据以确立问题的范围

**(c)** 承认已经采取的行动，表明你了解背景

**(d)** 保持序言部分简洁——重点应放在执行性条款上

**(e)** 按逻辑顺序排列条款（历史背景→当前情况→关切表达）

### 执行性条款写作提示

**(a)** **要具体**："请求成员国每年向绿色气候基金捐款5000万美元"比"请求增加资金"更好。

**(b)** **要可操作**：每个执行性条款都应提出实际可以完成的事情。

**(c)** **要现实**：提出在联合国实际能力和权限范围内的行动。

**(d)** **考虑实施**：谁来做？如何做？何时做？需要什么资源？

**(e)** **解决资金问题**：优秀的决议会解释如何为提案提供资金。

**(f)** **包含问责制**：如何衡量成功？谁报告进展？

### 使用子条款

子条款将复杂的提案分解为可管理的部分：

**(a)** 使用子条款提供细节、示例或实施步骤

**(b)** 第一级：小写字母（a, b, c）

**(c)** 第二级：罗马数字（i, ii, iii）

**(d)** 不要超过两层嵌套——否则会难以跟进

## 为你的决议建立联盟

决议需要支持才能通过。建立联盟与撰写文件本身同样重要。

### 寻找伙伴

**(a)** **识别志同道合的国家**：寻找与你所代表国家立场或利益相同的代表。

**(b)** **考虑区域集团**：来自同一地区的国家在许多问题上通常一起投票。

**(c)** **寻找互补利益**：即使优先事项不同的国家，如果你的决议中包含对他们有利的内容，也可能支持你。

### 收集起草国和附议国

**(a)** **起草国**是主要作者。选择活跃、有影响力的代表，他们会帮助介绍和捍卫决议。

**(b)** **附议国**只是想看到决议被讨论——他们不必投票支持。在这里可以撒更大的网。

**(c)** 在自由磋商期间以清晰的说辞接触潜在支持者。

**(d)** 愿意纳入他人的想法以获得他们的支持。

### 合并决议

**(a)** 如果多个集团有类似的决议，考虑合并以获得更强的支持。

**(b)** 识别每份决议中最好的条款。

**(c)** 协商哪些想法纳入最终合并文件。

**(d)** 确保所有合并方都成为起草国。

## 常见错误避免

注意避免以下常见的决议写作错误：

### 语言模糊

**(a)** 避免像"采取适当行动"或"解决问题"这样的短语。

**(b)** 明确说明应该发生什么、由谁来做、何时做。

**(c)** 如果你无法具体解释某个条款的作用，它就不应该出现在决议中。

### 不切实际的提案

**(a)** 不要提出联合国实际上无法采取的行动（例如，强迫国家违背其意愿做某事）。

**(b)** 记住大多数联合国机构只能建议，不能强制。

**(c)** 考虑实际限制，如资金、政治可行性和主权。

### 忽视资金问题

**(a)** 优秀的决议会解决如何为提案提供资金的问题。

**(b)** 确定资金来源：现有联合国预算、自愿捐款、新机制。

**(c)** 对成员国实际会资助什么保持现实态度。

### 组织混乱

**(a)** 按逻辑将相关条款分组。

**(b)** 从基础性条款建立到更具体的条款。

**(c)** 确保执行性条款与序言性条款中提出的关切相关联。

### 格式错误

**(a)** 序言性条款：斜体开头短语，以逗号结尾

**(b)** 执行性条款：加粗/下划线开头短语，以分号结尾（最后一个条款以句号结尾）

**(c)** 执行性条款编号；序言性条款不编号

## 决议写作流程

遵循此流程高效撰写决议：

### 第一步：确定关键问题

**(a)** 需要解决哪些主要问题？

**(b)** 已经做了什么？还有哪些差距？

**(c)** 你所代表的国家想要实现什么？

### 第二步：头脑风暴解决方案

**(a)** 生成多种可能的方法。

**(b)** 从不同角度考虑解决方案（经济、政治、人道主义等）。

**(c)** 思考其他国家会支持什么。

### 第三步：先起草执行性条款

**(a)** 从你最有力的提案开始。

**(b)** 使每个条款具体且可操作。

**(c)** 添加子条款说明实施细节。

### 第四步：添加序言性条款

**(a)** 为你的执行性条款提供背景。

**(b)** 引用相关的过去行动和当前关切。

**(c)** 保持这部分简洁。

### 第五步：争取支持

**(a)** 与潜在联盟伙伴分享你的草案。

**(b)** 纳入反馈和额外的想法。

**(c)** 收集起草国和附议国。

### 第六步：修改和完善

**(a)** 仔细检查格式。

**(b)** 确保逻辑流畅和组织合理。

**(c)** 校对拼写和语法。

## 介绍你的决议

当你的决议提交讨论时，你需要介绍并为其辩护。

### 引言介绍

**(a)** 简要解释你的决议所针对的问题。

**(b)** 突出最重要的执行性条款。

**(c)** 解释为什么你的方法会有效。

### 回应问题

**(a)** 彻底了解你决议中的每个条款。

**(b)** 准备好解释每个提案背后的理由。

**(c)** 准备好回答可能的批评。

### 处理修正案

**(a)** 提前决定哪些条款是必要的，哪些是可协商的。

**(b)** 接受能改进决议的友好修正案。

**(c)** 准备好反对削弱你目标的非友好修正案。

## 常见问题

### 决议应该写多长？

没有固定长度，但质量比数量更重要。包含足够的条款来全面解决问题，但不要用不必要的内容填充。大多数有效的决议有5-8个序言性条款和8-15个执行性条款。

### 决议可以有多个起草国吗？

可以，但有限制（通常是委员会人数的5%，向上取整）。战略性地选择起草国——他们应该是积极的参与者，会帮助介绍和捍卫决议。

### 起草国和附议国有什么区别？

起草国是主要作者，对决议内容负责。附议国只是想看到决议被讨论——他们之后可以投反对票。成为附议国并不意味着认可。

### 如何在类似的执行性短语之间做选择？

根据你想要的行动强度来选择。"要求"比"敦促"更强，"敦促"比"鼓励"更强。将短语与条款的紧迫性和重要性相匹配。

### 如果我的决议失败了怎么办？

失败是学习的机会。分析为什么代表们投了反对票——是内容问题、政治问题还是其他原因？将这种洞察用于你的下一份决议。
